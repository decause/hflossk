Humanitarian Open Source Software

Initial Project Status Report

by Justin Lewis (with tweaks by Professor Shein)

 

All teams will be expected to give a presentation during the Pythonistas meeting next Thursday night (6:30-9PM.

 

These presentations should be published in at least one member’s blog before the meeting on Thursday.  Each team member should make a post that states the milestones to which he or she has committed, and everyone will be expected to track his or her own progress towards those milestones.


The following is a guideline for the presentations:

 

  • Quick Project Description and current status.
    • What is the project?
    • What was the condition of the project when you entered?
    • Give a brief assessment about the code, documentation, and community base of the project as you found it

 

  • Milestones and Goals for the quarter
    • What are your group accomplishment goals?
    • What milestones are you committing to?
    • Division of labor (who will be responsible for each task)
    • Any potential milestones you would like to investigate if time permits
    • What has your team done so far?

 

  • Is a release in the scope of this class?  Why or why not?

 

  • A Code Discussion
    • By this time you should be familiar with the code base if not directly involved with it
    • Any code confusing, anything interesting, tricky, or just have questions?
      • These meetings may have some python enthusiast, if you have anything you need help on or want to show off, this is the time to do it.
      • PLAN to bring at least one unresolved code issue to discuss
      •  
  • Give printed hard copy of Milestones, Goals & Division of Labor to Professor Shein
